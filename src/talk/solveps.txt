问题分析

从你的请求日志中看到：

json
{
  "image_base64": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAAAQABAA..."
}
请求格式是完全正确的！ 这证明：

✅ 字段名正确：用的是 image_base64
✅ 前缀正确：有 data:image/jpeg;base64,
✅ Base64格式正确：以 /9j/ 开头（JPEG标准开头）
🎯 问题根源

虽然前端请求格式正确，但问题出在后端转发给阿里云/ShowAPI时。从返回的 showapi_res_body: {…} 可以看出，请求到达了第三方服务但被拒绝了。

最可能的原因是图片的Base64字符串中包含了换行符。在你的日志中可以看到：

text
"data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAAAQABAA..."
注意 ...ABAA 后面有换行，这在JSON字符串中可能被转义，导致第三方服务解析失败。

🛠️ 解决方案

1. 前端修复方案（立即生效）

修改你的 fileToBase64 函数，确保Base64字符串是干净的：

javascript
// search.vue 中修改
fileToBase64(file) {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.readAsDataURL(file);
    reader.onload = () => {
      let base64 = reader.result;
      
      // 关键修复：清理base64字符串
      base64 = base64
        .replace(/\s/g, '')           // 移除所有空白字符（包括换行符、空格）
        .replace(/[\n\r]/g, '')        // 额外移除换行符
        .trim();                        // 去除首尾空格
      
      console.log('清理后的base64长度:', base64.length);
      console.log('是否有换行符:', base64.includes('\n'));
      
      resolve(base64);
    };
    reader.onerror = error => reject(error);
  });
}
